# 📊 Автоматический сканер сигналов Supply/Demand

## 🧠 Обзор проекта
По файлам данного проекта была выложена статья на хабр, также готовится полноценное youtube видео с разобором.
Данный проект представляет собой полностью автоматизированную систему анализа рынка криптовалют, которая находит торговые сигналы на основе концепции **Supply & Demand зон** и подтверждает их через **Price Action (pinbar)**.

👉Профессиональный трейдинг и алгоритмы в тг: https://t.me/crypto_maniacdt

👉Больше разборов тут - https://www.youtube.com/@crypto_maniacdt

Система в реальном времени:

* сканирует все торговые пары USDT на Bybit
* определяет ключевые зоны ликвидности (supply/demand)
* отслеживает возврат цены в эти зоны
* подтверждает вход через свечной паттерн
* отправляет сигнал в Telegram с графиком

Архитектурно это конвейер обработки данных (pipeline), состоящий из последовательных этапов анализа рынка.

---

## 🏗 Архитектура системы

Система состоит из следующих логических блоков:

1. Получение рыночных данных (Bybit API)
2. Расчёт волатильности (ATR)
3. Поиск зон Supply/Demand
4. Фильтрация и очистка зон
5. Проверка текущей цены относительно зон
6. Подтверждение сигнала через Price Action
7. Генерация графика
8. Отправка сигнала в Telegram

Каждый блок выполняет строго определённую задачу, обеспечивая стабильность и масштабируемость системы.

## 📡 Принцип работы

Система работает в бесконечном цикле:

1. Получает список всех USDT пар
2. Загружает исторические свечи
3. Рассчитывает ATR
4. Находит зоны спроса и предложения
5. Удаляет невалидные зоны
6. Выбирает ближайшие к цене
7. Проверяет 5-минутную свечу
8. При наличии pinbar формирует сигнал
9. Отправляет график в Telegram

---

## 📈 Логика стратегии

Стратегия основана на следующей модели:

> Цена → Зона → Реакция → Сигнал

### 🔵 Demand зона (лонг)

* цена возвращается в зону спроса
* формируется бычий pinbar
* ожидается рост

### 🔴 Supply зона (шорт)

* цена возвращается в зону предложения
* формируется медвежий pinbar
* ожидается падение

---

## 🕯 Подтверждение через Pinbar

Сигнал считается валидным только если:

* тело свечи маленькое
* есть длинный фитиль
* направление соответствует входу

Это фильтрует ложные входы.

## 🧩 Описание основных функций

### 📥 Получение данных

* `get_symbols()` — список торговых пар
* `get_klines()` — исторические свечи

---

### 🧠 Анализ зон

* `detect_rejection_zones()` — поиск зон через реакцию
* `is_zone_broken()` — проверка пробоя
* `remove_overlapping_zones()` — удаление пересечений

---

### 🎯 Сигналы

* `check_long_signal()` — поиск лонга
* `check_short_signal()` — поиск шорта
* `price_in_zone()` — проверка входа в зону

---

### 📊 Визуализация

* `generate_chart()` — построение графика с зонами

---

### 📤 Telegram

* `send_telegram()` — отправка сигнала

---

### 🔄 Основной цикл

* `process_symbol()` — обработка одного инструмента
* `main_loop()` — бесконечный запуск
